## Implementation Tasks

### Task 1: Fix dotenv export escaping (Finding 1)
- Escape `$` as `\$` and backtick as `` \` `` in the double-quote path
- Escape `\r` as `\r`
- Add integration test for value containing `$(cmd)` to verify no expansion
- **Docs:** Update README export section to note that dotenv output is safe for `source`

### Task 2: Filter PASSPHRASE_FD from editor env (Finding 2)
- Change `filterEnv` call in edit.go to also strip `WZVAULT_PASSPHRASE_FD`
- **Docs:** Update `edit` command Long description to mention env sanitization

### Task 3: SQLite hardening (Findings 3, 5, 6, 10, 13, 14)
All in `internal/store/store.go`:
- Add `PRAGMA busy_timeout=5000` after WAL mode
- Add `db.SetMaxOpenConns(1)` after sql.Open
- Pre-create DB file at 0600 before sql.Open (remove post-open Chmod)
- Set `syscall.Umask(0077)` before open, restore after (for WAL/SHM sidecar files)
- Fix `schema_version` to singleton pattern with `CHECK(id = 1)`
- Wrap `tx.Commit()` error
- **Docs:** Update `internal/store/doc.go` to mention busy_timeout and WAL mode behavior
- **Docs:** Update README Configuration section to note concurrent access is supported

### Task 4: Hoist ParseExpiry in list (Finding 4)
- Compute threshold once before the loop using `ParseExpiryAt(listExpiring, now)`

### Task 5: Init --force error logging (Finding 7)
- Log `os.Remove` errors as warnings to stderr instead of silently discarding
- **Docs:** Update `init` command Long description to clarify `--force` behavior (deletes old files)

### Task 6: Document expiry as advisory (Finding 8)
- **Docs:** Add note in `get` command Long description that expired secrets are still accessible
- **Docs:** Add note in `list` command Long description that `--expired`/`--expiring` are informational filters
- **Docs:** Update README Security Model section to state expiry is advisory, not enforced on read
- **Docs:** Update README Usage section (get) to mention expiry is not enforced

### Task 7: Transactional import (Finding 9)
- Add `BeginTx()`/`SetTx()` or wrap the import loop in a store-level transaction
- **Docs:** Update `import` command Long description to mention atomic import behavior

### Task 8: Low-priority polish (Findings 11, 12, 15, 16)
- Auto-append `/` to `--prefix-strip` if missing, log normalization to stderr
- Use `/dev/tty` for delete confirmation (consistent with set)
- Standardize public-key newline stripping to `strings.TrimSpace` across set.go, import.go, edit.go
- Add `test-integration` target to Makefile
- **Docs:** Update CONTRIBUTING.md to reference `make test-integration`

### Task 9: Tests, docs, and release
- Run all existing tests (must pass)
- Add integration test for dotenv `$` escaping
- Full lint, full test suite
- **Docs:** Update CHANGELOG.md with all v0.2.1 fixes
- Push, tag v0.2.1
